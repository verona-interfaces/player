Introduction

The Player is currently expected to be loaded into an iframe in the Testcenter. Both ends of this interface communicate via postMessage. The only data saved beyond the livetime of a Player is the data transfered in restorePoint

1. Communication squence

1.1 General course

After the Player loads it announces it's state by a ReadyNotification which is answered by the Testcenter with an DataTransfer including the unitDefinition containing the unit and all subunits supposed to be presented. The Player then announces the beginning of the assesment with a StartedNotification. Entries submitted by the assessee are transfered to the Testcenter via a ChangedDataTransfer.

1.2 Navigation

If the current page is supposed to change due to events from the Testcenter this is communicated via a NavigateToPage, if the event to change the page is originated in the Player it sends a PageNavigationRequest or a UnitNavigationRequest if the loaded unit should be changed.

2. Message Definition

The following messages are defined for Player ðŸ¡˜ Testcenter communication. No other messages MUST be send. The word specifies means that information contained in an expression was created by the sending instance. The word containes means the information was received form a third party (e.g. a database) and is only forwarded.

2.1 ReadyNotification Player ðŸ¡’ Testcenter

Announcing Load Complete and sending the major part of version of the interface this Player understands (see above: starting by publishing this document with 1)

{
  type: 'vo.FromItemPlayer.ReadyNotification';
  version: number;
}
  • type specifies the type of the message
  • version specifies the interfaces major version number required to communicate

2.2 DataTransfer Testcenter ðŸ¡’ Player

{
  type: 'vo.ToPlayer.DataTransfer';
  sessionId: string;
  unitDefinition: string;
  restorePoint?: string;
  unitNumber?: string;
  unitTitle?: string;
}
  • sessionId specifies the Testcenter ðŸ¡˜ Player session
  • unitDefinition containes the definition of all units and subunits
  • restorePoint containes a serialised JSON value containing the last state of this unit
  • unitNumber specifies the number of the unit in the assessment
  • unitTitle specifies title of the current unit

2.3 StartedNotification Player ðŸ¡’ Testcenter

{
  type: 'vo.FromPlayer.StartedNotification';
  sessionId: string;
  validPages?: string[];
  currentPage?: string;
  presentationComplete?: 'yes' | 'no';
  responsesGiven?: 'yes' | 'no' | 'all';
}
  • validPages specifies the current list of valid accessible pages
  • currentPage specifies the currently active page
  • presentationComplete specifies the current presentation state
  • responsesGiven specifies the response state of the unit

2.4 ChangedDataTransfer Player ðŸ¡’ Testcenter

{
  type: 'vo.FromPlayer.ChangedDataTransfer';
  sessionId: string;
  validPages?: string[];
  currentPage?: string;
  restorePoint?: string;
  response?: string;
  responseConverter?: string;
  responseComplete?: boolean;
  presentationComplete?: 'yes' | 'no';
  responsesGiven?: 'yes' | 'no' | 'all';
}
  • restorePoint specifies a serialised JSON value containing the current state of this unit
  • response currently unspecified
  • responseConverter currently unspecified
  • responseComplete currently unspecified

2.5 NavigateToPage Testcenter ðŸ¡’ Player

Requests that the player has to navigate to the given page. Completion is announced by the Player via ChangedDataTransfer.

{
  type: 'vo.ToPlayer.NavigateToPage';
  sessionId: string;
  newPage: string;
}
  • newPage specifies to which page to navigate and is one of the strings given via validPages.

2.6 PageNavigationRequest Player ðŸ¡’ Testcenter

The Player requests the Testcenter to check and send a NavigateToPage.

{
  type: 'vo.FromPlayer.PageNavigationRequest';
  sessionId: string;
  newPage: string;
}
  • newPage is the id of the requested page or #next/#previous.

2.7 UnitNavigationRequest Player ðŸ¡’ Testcenter

{
  type: 'vo.FromPlayer.UnitNavigationRequest';
  sessionId: string;
  navigationTarget: string;
}
  • navigationTarget is an sequence-ID of a certain unit or one of these: (unit) #next, #previous, #first, #last, (booklet) #end.

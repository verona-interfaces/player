This document defines the communication between a **Testcenter** and a **Player** to execute computer based assesments as offered for the German [VERA - Vergleichsarbeiten](https://www.iqb.hu-berlin.de/vera), or the [National Assessment Studies and IQB Trends in Student Achievement](https://www.iqb.hu-berlin.de/bt).
The defintion was proposed and is currently discussed.
## Introduction The **Player** is currently expected to be loaded into an `iframe` in the **Testcenter**. Both ends of this interface communicate via [`postMessage`](https://developer.mozilla.org/en-US/docs/Web/API/Window/x-doc-messaging). The only data saved beyond the livetime of a **Player** is the data transfered in `restorePoint`

Can I Use postMessage? Data on support for the postMessage feature across the major browsers from caniuse.com.

## Communication squence ### General course After the **Player** loads it announces it's state by a `ReadyNotification` which is answered by the **Testcenter** with an `DataTransfer` including the `unitDefinition` containing the unit and all subunits supposed to be presented. The **Player** then announces the beginning of the assesment with a `StartedNotification`. Entries submitted by the assessee are transfered to the **Testcenter** via a `ChangedDataTransfer`. ### Navigation If the current page is supposed to change due to events from the **Testcenter** this is communicated via a `NavigateToPage`, if the event to change the page is originated in the **Player** it sends a `PageNavigationRequest` or a `UnitNavigationRequest` if the loaded unit should be changed.
